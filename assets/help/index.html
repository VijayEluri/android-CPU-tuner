cpu tuner help

CPU tuner uses a concept of triggers, profiles and virtual governors. Triggers are activated on your phone entering a predefined state. They then apply a predefined profile. The states are defined by a battery level and the phone's status (locked, active, on power, phone call in progress or even battery hot). Profiles are essentially a set of policies that are applied to the phone and change the state of services as well as the governor. Virtual governors are an abstraction of the CPU governors and their parameters. While doing so, CPU tuner runs event-driven. I.e. it does not run constantly, thus creating only minimal system load on it's own. How should I use this to save battery? This help also includes a FAQ which you might find helpful/interesting.
